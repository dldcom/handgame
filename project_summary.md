# 🤖 AI 수어 연구소 Phase 1 데이터 수집 웹 기능 개선 보고서

본 보고서는 초등학교 4학년 특수교육 대상 학생들을 위한 'AI 데이터 레이블러 양성 프로젝트'의 Phase 1(데이터 수집 웹 인터페이스) 고도화 및 최적화 작업 내역을 담고 있습니다.

기존 코드베이스의 한계(단일 손 감지, 위치 정보 유실, 데이터 오염, 성능 병목)를 기술적/UX적으로 극복하여 완벽한 데이터 수집 환경을 구축하였습니다.

---

## 🛠️ 주요 개선 사항 및 특징

### 1. 👐 양손 추적 및 다중 스켈레톤 인쇄
- **기존**: 최대 1개의 손만 감지하여 데이터를 수집함.
- **개선**: MediaPipe Hands의 `numHands` 옵션을 `2`로 확장하여 양손 동작을 완벽히 인식하며, 메인 캔버스와 AI 전용 미니 캔버스(`AI's EYE`) 모두에 두 손의 뼈대선이 동시에 렌더링되도록 드로잉 엔진을 다중 루프로 재설계했습니다.

### 2. 👤 신체 정렬 가이드 실루엣 (Body Silhouette Guide) 탑재
- **문제점**: 학생들의 앉은키나 카메라 각도에 따라 손의 물리적 Y축 절대 좌표가 흔들려 AI가 수어의 높낮이 위치(머리 위, 가슴 앞, 배 앞)를 올바르게 학습하지 못하는 치명적인 취약점이 존재했습니다.
- **해결책**:
  - 카메라 화면 상단에 SVG 기반의 **야광 노란색 점선 신체 실루엣(머리 원, 어깨선) 가이드**를 오버레이로 구현하였습니다.
  - 마스크(`mask`) 정의를 적용하여 실루엣 정렬 영역 안쪽은 깨끗하게 보이고, 외곽 영역은 어둡게 가려져 학생들이 직관적으로 몸을 쉽게 맞출 수 있는 UI를 제공합니다.
  - 이를 통해 **화면 상단 = 머리 위**, **화면 중앙 = 가슴 앞**, **화면 하단 = 배 앞**으로 좌표계를 완벽히 고정(Calibration)하였습니다.

### 3. 📐 위치 보존형 1:1 크롭 비례 투영 매핑
- **기존**: 감지된 각 손을 무조건 손 Bounding Box 기준 캔버스 중앙으로 늘리고 확대(정규화)하여, 수어에서 가장 중요한 손의 물리적 공간 높낮이 정보가 완전히 유실되었고 양손이 정중앙에서 겹치는 버그가 있었습니다.
- **개선**:
  - 개별 손 정규화 방식을 중단하고, 640x480 등 원본 카메라의 1:1 중앙 크롭 박스 비율 전체를 224x224 도화지에 비례 투영 매핑하는 수식을 개발했습니다.
  - 신체 가이드 실루엣에 몸을 정렬한 상태로 손을 머리 위/가슴 앞/배 앞에 취하면, AI가 학습할 224x224 블랙 캔버스 상단/중앙/하단에 손의 높이와 크기가 정확히 기록되어 절대 공간 좌표를 학습할 수 있습니다.

### 4. ⏱️ 3초 카운트다운 & 2초 버스트(Burst) 자동 녹화 시스템
- **문제점**: 수집 버튼을 계속 켜두면, 수어 동작을 마친 뒤 손을 원래대로 내리는(제자리로 복귀하는) 불필요한 자세까지 전부 해당 단어로 잘못 수집되는 **데이터 오염(라벨 노이즈)** 문제가 심각했습니다.
- **해결책**:
  - 버튼 클릭 시 카메라 한가운데 거대한 카운트다운(`3 ➔ 2 ➔ 1 ➔ 시작! 🎬`) 애니메이션을 노출하여 학생이 첫 자세를 예비할 수 있는 시간을 확보합니다.
  - `시작!` 신호와 함께 **정확히 2초(8장 분량, 0.25초당 1장)** 동안만 녹화가 켜지며, 이때 카메라 하단에 **빨간색 실시간 충전 프로그레스 바**를 제공하여 녹화 상태를 가시화합니다.
  - 2초가 지나면 캡처 수집이 **자동으로 정지**되어 대기 상태로 돌아갑니다.
- **자동 안전 필터**: 동작이 완료된 후 손을 화면 밖이나 어깨 밑으로 내리면, 손 감지 조건(`hands.length > 0`)이 꺼져서 불필요한 복귀 포즈가 단 1장도 서버에 기록되지 않도록 완벽히 방어합니다.

### 5. ⚡ 렉 방지 쓰로틀링(Throttling) 및 FPS 최적화
- **문제점**: 60 FPS로 가동되는 애니메이션 렌더링 루프 내에서, 매 프레임마다 메모리에 새 캔버스를 올리고 이미지 변환(PNG Blob 생성)을 수행하여 CPU/GPU 부하가 걸려 화면이 뚝뚝 끊기는 심각한 렉이 발생했습니다.
- **해결책**:
  - 이미지 캡처/변환 연산을 60 FPS에서 실제 업로드 주기인 **250ms(초당 4회) 간격으로 원천 쓰로틀링 제어**하여 연산 주기를 15배 감축했습니다.
  - 이를 통해 녹화 중에도 화면 버벅임이나 끊김 현상이 전혀 없이 **매끄러운 실크 수준의 60 FPS 카메라 구동**을 구현했습니다.

### 6. 📝 표준 한국수어(KSL) 설명 교정 및 클라우드 경로 개선
- **가이드 교정**: 국립국어원 표준 수어 정의에 부합하도록 오류가 많았던 기존 수어 설명 가이드 팁을 100% 정석으로 전면 교정 및 교체하였습니다.
- **한글 경로 차단 및 자동 맵핑**: 
  - Supabase Storage 버킷 경로 상에 한글명(예: `skeletons/안녕하세요/...`)이 담기면 주소 인코딩 에러(`Invalid key`)로 서버가 차단을 먹는 현상을 방지하기 위해, 저장 시 안전한 영문 ID 폴더(`skeletons/hello/...`)를 매핑해 저장하도록 변경했습니다.
  - 교사용 일괄 다운로드 스크립트(`download_dataset.js`) 실행 시에는 자동으로 로컬 컴퓨터에 한글 명칭 폴더(`안녕하세요` 등)로 복원하여 생성 및 다운로드해 주도록 지능화 설계했습니다.
